# Continuity

Continuity is one of the defining parts of the Iris workflow.

Once a user accepts a generated result, Iris can push that decision further through the surrounding cut so the edit feels authored instead of isolated.

## What continuity means in Iris

In this project, continuity is about preserving a chosen visual interpretation across time.

That usually means:

- keeping the accepted subject treatment coherent
- helping the result survive motion and framing changes
- reducing the feeling that a single frame was changed in isolation

## How entity detection works

After a user accepts a variant, Iris triggers entity detection. Gemini analyzes the accepted edit and searches the rest of the project timeline for appearances of the same entity — a person, object, or visual element — in other frames.

Each discovered appearance is tracked as an **EntityAppearance** with:

- the frame timestamp where the entity appears
- a bounding box around the entity in that frame
- a confidence score indicating how certain the match is

This gives Iris a map of everywhere the edited subject shows up across the clip.

## The propagation flow

Once entity appearances are known, the user can propagate the accepted edit to all of them:

1. **Fan out** — `POST /api/propagate` takes the source variant, entity id, and prompt, then creates a sub-job for each appearance.
2. **Generate per appearance** — each sub-job runs a localized generation at that appearance's frame and bounding box, using the source variant as reference.
3. **Review individually** — the user reviews each propagation result one at a time. Results are not auto-applied unless `auto_apply` was set.
4. **Apply selectively** — `POST /api/propagate/{job_id}/apply/{result_id}` commits a single result to the timeline.

This keeps the user in control. A propagation might produce great results for close-up frames but poor ones for distant shots, so per-result review matters.

## The continuity dashboard

In pro mode, the continuity dashboard shows:

- **Discovery status** — whether entity detection has run, how many appearances were found, and their confidence distribution
- **Propagation progress** — how many appearances have been generated, how many are pending, and overall completion percentage
- **Per-appearance results** — individual review cards showing the original frame, the propagated variant, and accept/reject controls

## Supporting product features

- Accepted edit flow in the inspector
- Continuity dashboard and status tracking
- Multi-frame generation session helpers
- Project-level entity awareness

## Why it matters

A single good frame is not enough for believable video editing.

The accepted result needs to carry through enough of the surrounding sequence that the viewer reads it as an intentional editorial decision.
