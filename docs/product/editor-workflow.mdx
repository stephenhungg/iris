## Why this workflow exists

Full-scene generation is too global for many real edits.

Traditional editing is precise, but every meaningful visual change is still manual.

Iris exists between those models. It is designed for:

- Local edits instead of total scene replacement
- Selection and timing instead of broad prompting
- Multiple options instead of one irreversible output
- Editorial choice instead of blind automation

## Editing modes

Iris has two editing modes. Toggle between them via the view menu or the mode pill in the top bar.

### Vibe mode

A simplified 60/40 layout with the video player on the left and an agent chat panel on the right. Users describe edits in natural language and the agent handles the rest — it uses Gemini function calling to analyze the video, identify regions, generate edits, and present variants for review.

Vibe mode includes a video scrubber below the player for navigating the clip and tool call visualization in the chat so you can see exactly what the agent is doing at each step.

This is the best starting point for users who want to edit conversationally without manually selecting regions or configuring parameters.

### Pro mode

The full editor with a library sidebar, inspector panel (AI/basic/info tabs), timeline, and continuity dashboard. Pro mode gives more granular control over individual edits — manual bounding box selection, direct generation parameter tuning, and frame-level timeline management.

Use pro mode when you need precise control over timing, region selection, or want to manage continuity propagation manually.

## Main surfaces in the editor

### Library

The library holds the media available to the current project, including uploads and generated assets.

### Preview

The preview is where users inspect frames, scrub through the clip, and make a visual selection.

### Inspector

The inspector turns the active selection into an actionable AI editing flow. It handles generation controls, accepted edit flow, and continuity follow-through.

### Timeline

The timeline is where accepted edits stop being isolated outputs and become part of an editorial sequence.
