# Backend API

This page is a practical route reference, not a formal OpenAPI spec. All routes are prefixed with `/api`.

## Health and status

### `GET /api/health`

Returns server liveness status.

### `GET /api/me`

Returns the currently authenticated user profile.

## AI and Agent Operations

### `GET /api/ai/health`

Returns AI provider reachability and configuration status.

### `GET /api/ai/timeline`

Returns a timeline of recent AI operations, including timing and status.

### `GET /api/ai/stream`

Establishes a Server-Sent Events (SSE) connection to stream real-time AI operation events and agent chat updates. This endpoint provides live feedback on ongoing AI tasks, such as prompt processing, generation progress, and agent responses.

**Response:** A stream of JSON objects, each representing an event.

```json
// Example event structure (may vary based on event type)
{
  "event": "agent_message",
  "data": {
    "id": "event-123",
    "timestamp": "2023-10-27T10:00:00Z",
    "type": "text",
    "content": "Analyzing video segment for continuity..."
  }
}
```
