# Backend API

This page is a practical route reference, not a formal OpenAPI spec. All routes are prefixed with `/api`.

## Health and status

### `GET /api/health`

Returns server liveness status.

### `GET /api/me`

Returns the currently authenticated user profile.

## AI and Agent Operations

### `GET /api/ai/health`

Returns AI provider reachability and configuration status.

### `GET /api/ai/timeline`

Returns a timeline of recent AI operations.

### `GET /api/ai/stream`

Returns a Server-Sent Events (SSE) stream of real-time AI operation events.

### `POST /api/agent/chat`

Initiates or continues an agent chat session.

**Request body:**

```json
{
  "project_id": "string",
  "message": "string",
  "conversation_id": "string",
  "history": [
    {
      "role": "user",
      "content": "string"
    },
    {
      "role": "model",
      "content": "string"
    }
  ],
  "playhead_ts": "number | null",
  "duration": "number | null",
  "bbox": {
    "x": "number",
    "y": "number",
    "w": "number",
    "h": "number"
  } | null
}
```

**Response:**

Server-Sent Events (SSE) stream of agent responses, tool calls, and results.

### Agent Tools

The agent can call the following functions:

#### `get_timeline`

Returns the current state of the video timeline for the active project.

**Parameters:**

```json
{
  "project_id": "string"
}
```

#### `preview_frame`

Generates a preview image for a specific timestamp on the timeline.

**Parameters:**

```json
{
  "project_id": "string",
  "timestamp": "number"
}
```

#### `preview_strip`

Generates a strip of preview images for a given segment of the timeline.

**Parameters:**

```json
{
  "project_id": "string",
  "start_ts": "number",
  "end_ts": "number"
}
```

#### `analyze_video`

Analyzes a video segment for content, entities, or scene changes.

**Parameters:**

```json
{
  "project_id": "string",
  "start_ts": "number",
  "end_ts": "number"
}
```

#### `identify_region`

Identifies and describes the content within a specified bounding box.

**Parameters:**

```json
{
  "project_id": "string",
  "timestamp": "number",
  "bbox": {
    "x": "number",
    "y": "number",
    "w": "number",
    "h": "number"
  }
}
```

#### `generate_edit`

Generates new video variants based on a prompt and a timeline segment.

**Parameters:**

```json
{
  "project_id": "string",
  "prompt": "string",
  "start_ts": "number",
  "end_ts": "number",
  "bbox": {
    "x": "number",
    "y": "number",
    "w": "number",
    "h": "number"
  } | null
}
```

#### `get_job_status`

Retrieves the current status of a video generation job.

**Parameters:**

```json
{
  "job_id": "string"
}
```

#### `wait_for_job`

Block until a generation job finishes (status=done) or a timeout elapses. This is typically called immediately after `generate_edit` so the variants are in hand before the agent replies.

**Parameters:**

```json
{
  "job_id": "string",
  "timeout_s": "number" // Optional, default 180
}
```

**Response:**

Returns the job status object, including a `waited_s` field indicating how long the call blocked.

#### `accept_variant`

Accept a generated variant, applying it to the timeline.

**Parameters:**

```json
{
  "job_id": "string",
  "variant_index": "integer"
}
```

#### `snapshot_timeline`

Creates a snapshot of the current timeline state, allowing for easy reverts.

**Parameters:**

```json
{
  "project_id": "string"
}
```

#### `revert_timeline`

Reverts the timeline to a previously saved snapshot.

**Parameters:**

```json
{
  "project_id": "string",
  "snapshot_id": "string"
}
```

#### `score_variant`

Scores a generated variant based on quality, relevance, or other criteria.

**Parameters:**

```json
{
  "job_id": "string",
  "variant_index": "integer"
}
```

#### `score_continuity`

Analyzes the continuity of an edit across adjacent frames or segments.

**Parameters:**

```json
{
  "project_id": "string",
  "start_ts": "number",
  "end_ts": "number"
}
```

#### `remix_variant`

Generates new variants based on an existing variant and a new prompt.

**Parameters:**

```json
{
  "job_id": "string",
  "variant_index": "integer",
  "prompt": "string"
}
```

#### `split_segment`

Splits a video segment at a specified timestamp.

**Parameters:**

```json
{
  "project_id": "string",
  "timestamp": "number"
}
```

#### `trim_segment`

Trims a video segment to new start and end timestamps.

**Parameters:**

```json
{
  "project_id": "string",
  "segment_id": "string",
  "new_start_ts": "number",
  "new_end_ts": "number"
}
```

#### `delete_segment`

Deletes a video segment from the timeline.

**Parameters:**

```json
{
  "project_id": "string",
  "segment_id": "string"
}
```

#### `color_grade`

Applies a color grading preset or custom adjustments to a segment.

**Parameters:**

```json
{
  "project_id": "string",
  "segment_id": "string",
  "preset": "string" // e.g., "cinematic", "vibrant"
}
```

#### `export_video`

Exports the current timeline as a video file.

**Parameters:**

```json
{
  "project_id": "string",
  "format": "string" // e.g., "mp4", "webm"
}
```

#### `get_export_status`

Retrieves the status of a video export job.

**Parameters:**

```json
{
  "job_id": "string"
}
```
