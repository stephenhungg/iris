# Backend API

This page is a practical route reference, not a formal OpenAPI spec. All routes are prefixed with `/api`.

## Health and status

### `GET /api/health`

Returns server liveness status.

### `GET /api/me`

Returns the currently authenticated user profile.

### `GET /api/ai/health`

Returns AI provider reachability and configuration status.

### `GET /api/ai/timeline`

Returns a timeline of recent AI operations.

## Preview Endpoints

These endpoints provide low-resolution previews of media, resolving timestamps across original and generated variant media.

### `GET /api/preview/frame`

Returns a low-resolution image preview for a specific frame at a given timestamp within a project's timeline.

**Query Parameters:**
- `project_id` (string, required): The ID of the project.
- `timestamp` (float, required): The timestamp in seconds for the desired frame.

**Response:**
- `200 OK`: Returns a URL to the preview image.
```json
{
  "url": "https://example.com/path/to/preview_frame.jpg"
}
```

### `GET /api/preview/strip`

Returns a low-resolution video strip preview for a short duration around a given timestamp.

**Query Parameters:**
- `project_id` (string, required): The ID of the project.
- `timestamp` (float, required): The center timestamp in seconds for the desired strip.
- `duration` (float, optional, default: 2.0): The duration in seconds of the strip.

**Response:**
- `200 OK`: Returns a URL to the preview video strip.
```json
{
  "url": "https://example.com/path/to/preview_strip.mp4"
}
```

### `GET /api/preview/range`

Returns a low-resolution video preview for a specified time range, stitching together original and variant media as needed.

**Query Parameters:**
- `project_id` (string, required): The ID of the project.
- `start_ts` (float, required): The start timestamp in seconds of the range.
- `end_ts` (float, required): The end timestamp in seconds of the range.

**Response:**
- `200 OK`: Returns a URL to the preview video range.
```json
{
  "url": "https://example.com/path/to/preview_range.mp4"
}
```

## Scoring Endpoints

These endpoints provide AI-driven scoring and analysis of generated media variants and continuity.

### `POST /api/score`

Scores a single generated media variant based on a given prompt or criteria.

**Request Body:**
```json
{
  "project_id": "string",
  "segment_id": "string",
  "variant_id": "string",
  "prompt": "string"
}
```

**Response:**
- `200 OK`: Returns a score or analysis result for the variant.
```json
{
  "score": 0.85,
  "feedback": "The variant effectively addresses the prompt, with minor visual artifacts."
}
```

### `POST /api/score/compare`

Compares two generated media variants and ranks them based on a given prompt or criteria.

**Request Body:**
```json
{
  "project_id": "string",
  "segment_id": "string",
  "variant_id_1": "string",
  "variant_id_2": "string",
  "prompt": "string"
}
```

**Response:**
- `200 OK`: Returns a comparison result, indicating which variant is preferred and why.
```json
{
  "preferred_variant_id": "string",
  "comparison_notes": "Variant 1 has better lighting and color grading, while Variant 2 shows some blur."
}
```

### `POST /api/score/continuity`

Analyzes the continuity of an accepted edit across adjacent frames or segments.

**Request Body:**
```json
{
  "project_id": "string",
  "segment_id": "string",
  "entity_job_id": "string"
}
```

**Response:**
- `200 OK`: Returns an analysis of continuity, potentially highlighting areas of inconsistency.
```json
{
  "continuity_score": 0.92,
  "analysis": "Continuity is largely maintained, with a slight jump detected at 15.2s."
}
```
