# Backend API

This page is a practical route reference, not a formal OpenAPI spec. All routes are prefixed with `/api`.

## Health and status

### `GET /api/health`

Returns server liveness status.

### `GET /api/me`

Returns the currently authenticated user profile.

### `GET /api/ai/health`

Returns AI provider reachability and configuration status.

### `GET /api/ai/timeline`

Returns a timeline of recent AI operations.

## Preview

### `GET /api/preview/{pid}/frame`

Returns a single frame from a project at a given timestamp.

**Query Parameters:**
- `ts` (float): The timestamp of the frame to retrieve.

### `GET /api/preview/{pid}/strip`

Returns a strip of frames for a given time range and frame rate.

**Query Parameters:**
- `start` (float): The start timestamp of the range.
- `end` (float): The end timestamp of the range.
- `fps` (float, optional): Frames per second for the strip. Defaults to `1.0`.

### `GET /api/preview/{pid}/range`

Returns a preview of a time range within a project.

**Query Parameters:**
- `start` (float): The start timestamp of the range.
- `end` (float): The end timestamp of the range.

## Timeline Surgery

### `POST /api/timeline/{pid}/split`

Splits a timeline segment at a specified timestamp.

**Request Body:**
```json
{
  "segment_id": "string",
  "split_ts": 0.0
}
```

### `POST /api/timeline/{pid}/trim`

Trims a timeline segment to new start and end timestamps.

**Request Body:**
```json
{
  "segment_id": "string",
  "new_start_ts": 0.0,
  "new_end_ts": 0.0
}
```

### `POST /api/timeline/{pid}/delete`

Deletes a timeline segment.

**Request Body:**
```json
{
  "segment_id": "string"
}
```

### `POST /api/timeline/{pid}/reorder`

Reorders timeline segments within a project.

**Request Body:**
```json
{
  "segment_ids": ["string", "string"],
  "order": [0, 1]
}
```

### `POST /api/timeline/{pid}/snapshot`

Creates a snapshot of the current project timeline.

### `POST /api/timeline/{pid}/revert`

Reverts the project timeline to a specified snapshot.

**Request Body:**
```json
{
  "snapshot_id": "string"
}
```

## Color Grading

### `POST /api/segments/{sid}/grade`

Applies color grading adjustments to a specific segment.

**Request Body:**
```json
{
  "adjustments": {
    "brightness": 0.0,
    "saturation": 0.0
  }
}
```

### `POST /api/segments/{sid}/grade/preview`

Previews color grading adjustments for a specific segment without applying them.

**Request Body:**
```json
{
  "adjustments": {
    "brightness": 0.0,
    "saturation": 0.0
  }
}
```

### `POST /api/grade/match`

Matches the color grading of a source segment to a reference segment.

**Request Body:**
```json
{
  "source_id": "string",
  "reference_id": "string"
}
```

## Scoring

### `POST /api/score`

Scores a single generated variant based on a comparison target.

**Request Body:**
```json
{
  "variant_id": "string",
  "compare_to": "string" // e.g., "prompt"
}
```

### `POST /api/score/compare`

Compares and scores multiple variants against each other.

**Request Body:**
```json
{
  "variant_ids": ["string", "string"]
}
```

### `POST /api/score/continuity`

Scores the overall continuity of a project.

**Request Body:**
```json
{
  "project_id": "string"
}
```

## Remix & Batch

### `POST /api/remix`

Remixes an existing variant with a modifier prompt.

**Request Body:**
```json
{
  "variant_id": "string",
  "modifier_prompt": "string",
  "preserve_composition": true
}
```

### `POST /api/batch/generate`

Submits a request to generate multiple edits in a batch.

**Request Body:**
```json
{
  "edits": [] // Array of edit objects
}
```

### `POST /api/batch/accept`

Accepts multiple batch generation results.

**Request Body:**
```json
{
  "accepts": [] // Array of accept objects
}
```
